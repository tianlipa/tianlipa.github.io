LeetCode Heap/Priority Queue 题解

发表于 2026-04-16 分类于 LeetCode 阅读次数: 本文字数: 837 阅读时长 ≈ 3 分钟

考虑了一下决定尝试换到 python.

Kth Largest Element In a Stream#

数据流中的第 K 大元素

注意是第 k 大, 所以搞个小顶堆, 一直 pop 到只剩 k 个元素, 返回堆顶元素即可.

Last Stone Weight#

最后一块石头的重量

Python 默认只有小顶堆, 所以可以把所有数字取相反数再存进去实现大顶堆.

K Closest Points to Origin#

最接近原点的 K 个点

Python 比较元组会先比较第一个元素.

也可以重写 __lt__ 方法或者用 dataclass , 但没有这么方便.

Kth Largest Element In An Array#

数组中的第K个最大元素

堆可以做, 但复杂度是 O (NlogK), 我非常想知道是谁把这题放到堆的分类里的.

答案是快速选择.

Task Scheduler#

任务调度器

操作系统还在追我.

贪心, 优先完成剩余次数最多的任务, 用双向队列记录冷却时间, 冷却完成之后把它放回堆里.

或者, 考虑总时长是由最频繁的任务决定的, 例如 n=2, 最频繁的任务是 A, 则先排好 A:

若有多个和 A 一样频繁的任务, 会影响最后一组的大小, 其余直接放入空位中, 若空位放不下则任意安排都不会影响总时长. 所以可以直接算.

Find Median From Data Stream#

数据流的中位数

一个大顶堆放较小的一部分数, 一个小顶堆放较大的一部分数.

其实 push 逻辑可以优化, 但我懒得改了.
